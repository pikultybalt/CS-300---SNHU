#CS 300 Portfolio Reflection – Module Eight Journal
Overview

This portfolio submission includes my work from both major projects in CS 300:

An analysis of run-time and memory usage for different data structures
A working C++ program that loads, sorts, and prints Computer Science courses in alphanumeric order

These artifacts demonstrate my understanding of data structures, algorithm efficiency, and program design.

#Reflection

#What problem was I solving in these projects?
The main problem I was solving was how to efficiently store, organize, and retrieve course data from a file. This included displaying courses in sorted order and allowing users to search for specific course information along with prerequisites. The challenge was choosing the right data structure to manage the data effectively while maintaining good performance.

#How did I approach the problem? Why are data structures important?
I approached the problem by first understanding how the data would be used—loading from a file, searching, and sorting. Then I compared different data structures like vectors, hash tables, and binary search trees. Data structures are important because they directly impact how fast and efficiently a program runs. For example, a binary search tree allows faster searching compared to a simple list, especially as the dataset grows.

#How did I overcome roadblocks?
One of the biggest challenges I faced was correctly implementing logic for parsing the file and handling edge cases like missing or improperly formatted data. I also struggled with understanding how to properly structure my program for searching and sorting. I overcame these issues by breaking the problem into smaller steps, testing frequently, and revisiting course materials to better understand the concepts.

#How has this project changed how I design software?
This project helped me realize the importance of planning before coding. Instead of jumping straight into writing code, I now think more about the structure of the program and which data structure is best suited for the problem. It also showed me how important efficiency is when designing software.

#How has this project improved my coding style (maintainability, readability, adaptability)?
My code has become more organized and easier to read. I now use clearer variable names, consistent formatting, and better separation of logic into functions. I also understand how to write code that can be easily updated or expanded in the future, which makes it more adaptable. Adding comments has also helped make my code more understandable for others.
